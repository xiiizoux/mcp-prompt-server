MCP Prompt Server API

提供提示词管理功能的 API 服务

API 概览

MCP Prompt Server API 提供了一系列用于管理和使用提示词的端点。这些端点允许您获取提示词列表、查看提示词详情、创建新提示词等。

所有 API 响应都遵循以下格式：

{
  "status": "success" | "error",
  "data": [响应数据] | null,
  "message": "响应消息" | null
}

POST /api/get_prompt_names

获取所有可用的提示词名称列表。

请求示例：

curl -X POST https://your-app.vercel.app/api/get_prompt_names

响应示例：

{
  "status": "success",
  "data": ["general_assistant", "code_assistant", "writing_assistant"]
}

POST /api/get_prompt_details

获取特定提示词的详细信息。

请求参数：

{
  "name": "提示词名称"
}

请求示例：

curl -X POST https://your-app.vercel.app/api/get_prompt_details \
  -H "Content-Type: application/json" \
  -d '{"name":"code_assistant"}'

响应示例：

{
  "status": "success",
  "data": {
    "name": "code_assistant",
    "description": "代码助手提示词",
    "category": "编程",
    "tags": ["代码", "编程", "开发"],
    "messages": [...]
  }
}

POST /api/reload_prompts

重新加载所有提示词。

请求示例：

curl -X POST https://your-app.vercel.app/api/reload_prompts

响应示例：

{
  "status": "success",
  "message": "已重新加载 3 个提示词"
}

POST /api/get_prompt_template

获取创建新提示词的 YAML 模板。

请求示例：

curl -X POST https://your-app.vercel.app/api/get_prompt_template

响应示例：

{
  "status": "success",
  "data": "name: prompt_name\ndescription: 提示词的描述\n..."
}

POST /api/create_prompt

创建新的提示词。

请求参数：

{
  "name": "提示词名称",
  "content": "YAML格式的提示词内容"
}

请求示例：

curl -X POST https://your-app.vercel.app/api/create_prompt \
  -H "Content-Type: application/json" \
  -d '{
    "name": "greeting_prompt",
    "content": "name: greeting_prompt\ndescription: 问候提示词\n..."
  }'

响应示例：

{
  "status": "success",
  "message": "提示词 'greeting_prompt' 创建成功",
  "data": { ... }
}

GET /api/health

健康检查端点，用于验证 API 是否正常运行。

请求示例：

curl https://your-app.vercel.app/api/health

响应示例：

{
  "status": "ok",
  "version": "1.0.0"
}
